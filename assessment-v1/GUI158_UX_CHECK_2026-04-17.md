## 3. Findings (Prioritized)

### F1 (High): Destructive actions have no confirmation or undo

Evidence:

- delete buttons are directly available in primary forms and list cards (`index.html`:62, `index.html`:166, `app.js`:673)
- delete handlers execute immediately (`app.js`:841, `app.js`:902)

User impact:

- accidental deletion risk is high in dense graph editing
- entity deletion can remove multiple incident relationships, amplifying loss cost

Recommendation:

- add confirm modal with explicit consequence copy
- add 6-second toast undo for relationship delete and entity delete (soft restore via cached payload)

### F2 (High): Selection interactions trigger full data refresh and can feel laggy

Evidence:

- selecting entity/relationship/focus always calls `refreshAll()` (`app.js`:783-803)
- `refreshAll()` refetches entities, relationships, graph, and selected relationship (`app.js`:771-776)

User impact:

- high interaction cost for exploratory clicking
- perceived instability on slower networks due global loading churn

Recommendation:

- split data loading paths:
  - local-only selection render for entity/relationship selection
  - graph-only fetch for focus changes
  - full refresh only for create/update/delete and explicit reload

### F3 (Medium): Critical interaction (`double-click to focus`) has no visible affordance

Evidence:

- graph nodes support `dblclick` to focus (`app.js`:1056)
- UI copy and controls do not teach this gesture

User impact:

- feature discoverability is low, especially for keyboard/touch users
- focus workflow appears slower than it needs to be

Recommendation:

- add inline graph hint: `Tip: Double-click a node to focus its neighborhood`
- add node context menu/button parity for non-double-click users

### F4 (Medium): Accessibility gap in loading semantics and focus visibility

Evidence:

- loading spinners are initialized with `aria-hidden="true"` (`index.html`:18,73,102)
- runtime loading toggles only `hidden` and does not update `aria-hidden` (`app.js`:223-229)
- no explicit `:focus`/`:focus-visible` treatment in stylesheet (`app.css`)

User impact:

- assistive technology may miss loading-state changes
- keyboard navigation affordance is weak and inconsistent

Recommendation:

- synchronize `aria-hidden` with loading state
- add clear `:focus-visible` ring across button/input/select/textarea and list card actions

### F5 (Medium): Information hierarchy is dense during first-time use

Evidence:

- all three large panels carry similar visual weight from first render (`index.html`:11-170)
- initial empty states are text-heavy and distributed across multiple regions (`index.html`:27-47,139-144)

User impact:

- onboarding path is unclear: users must infer “where to start”
- first-run cognitive load remains high

Recommendation:

- add single “Start here” stepper banner in left panel:
  1. Create/select entity
  2. Add relationship
  3. Focus graph
- collapse advanced sections until an entity or relationship is selected

## 4. Proposed UX Slice (Implementation-Ready)

### Slice Name

**Safe + Fast Exploration Pass**

### Goals

- reduce accidental destructive errors
- reduce selection-to-feedback latency
- improve first-run task completion

### UX Spec

1. **Delete Confirmation + Undo**
- Show modal on delete with dynamic copy:
  - Entity: `Delete {name}? This also removes {incidentCount} connected relationships.`
  - Relationship: `Delete relationship {id}?`
- Buttons: `Cancel` (secondary), `Delete` (danger)
- After delete, show toast with `Undo` for 6 seconds.

2. **Interaction Performance Model**
- `selectEntity` and `selectRelationship`: local state update + `render()` only.
- `focusGraph`: fetch `/api/graph?entityId=...` only.
- Keep `refreshAll()` for mutations and `Reload everything`.

3. **Graph Discoverability + Input Parity**
- Add helper text under graph toolbar:
  - `Click to select. Double-click to focus. Use “Show full graph” to exit focus.`
- Add keyboard-accessible `Focus` action in selected-entity summary and relationship cards.

4. **A11y Baseline Hardening**
- Update loading indicators:
  - when visible: `aria-hidden=false`, optional `role=status` and label text
  - when hidden: `aria-hidden=true`
- Add focus styles:
  - 2px visible ring, minimum 3:1 contrast, no layout shift.

5. **First-Run Guidance**
- On empty graph, replace generic empties with guided CTA block:
  - headline: `Start your graph`
  - primary CTA: `Create first entity`
  - secondary hint: `Then connect entities from Relationship inspector`

## 7. Acceptance Criteria

1. Delete flows require explicit confirmation and provide undo.
2. Entity/relationship selection no longer triggers full multi-endpoint refresh.
3. Focus behavior remains correct while using graph-only fetches.
4. Graph focus gesture is visibly documented in UI copy.
5. Loading indicators expose correct accessibility semantics.
6. Keyboard users have visible focus rings on all interactive controls.
7. First-run empty state presents a single clear starting path.
