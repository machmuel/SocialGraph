## 3. Findings (Prioritized)

### F1 (High): Graph interaction was mouse-dependent

Evidence before fix:

- SVG node/link interactions depended on click or double-click only.
- No semantic keyboard affordance on graph elements.

User impact:

- keyboard and assistive-tech users could not reliably inspect relationships or focus entities.

Status: **Fixed in this heartbeat**

Implementation:

- Added keyboard-operable graph shells (`role=button`, `tabindex=0`, `aria-label`, Enter/Space activation).
- Added `F` keyboard shortcut on focused node shells to trigger graph focus.
- Added explicit interaction hint text near graph toolbar.

### F2 (High): Destructive actions had no guardrail

Evidence before fix:

- Entity/relationship delete actions executed immediately from UI controls.

User impact:

- accidental deletion risk in dense graph editing sessions.

Status: **Fixed in this heartbeat**

Implementation:

- Added confirmation dialogs for entity and relationship deletion with consequence-aware copy.

### F3 (Medium): Dynamic list copy reduced scan clarity

Evidence before fix:

- singular/plural labels always used singular (`incident`) in several list and summary views.

User impact:

- lower readability and increased cognitive friction during filtering/scanning.

Status: **Fixed in this heartbeat**

Implementation:

- Added shared `pluralize()` helper and applied it in entity cards, neighbor chips, and relationship summaries.

### F4 (Medium): Focus visibility for keyboard navigation was weak

Evidence before fix:

- no explicit global `:focus-visible` treatment.

User impact:

- poor keyboard orientation across dense controls.

Status: **Fixed in this heartbeat**

Implementation:

- Added consistent `:focus-visible` ring for form controls and emphasized focused graph links/nodes.

## 4. Change Summary (Shipped)

2. `index.html`

3. `app.css`

## 5. Remaining UX Risks (Next Slice)

1. Selection currently still triggers broad refresh paths in some flows; splitting local-selection render from network refresh would reduce interaction latency further.
2. Delete flow now confirms but still has no undo path; toast-based undo remains a high-value follow-up.
3. Graph uses radial static layout; large graphs may need zoom/pan and collision handling for readability.
